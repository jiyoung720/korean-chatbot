## 발견한 추가 한계: 질문 형태에 따른 검색 불안정성

이전 실험("세종대왕은 누구야?")에서는 한글 창제 청크가 score 0.48로
정확히 검색되었으나, 이번 평가셋 질문("세종대왕은 어떤 업적으로
유명한가?")으로는 완전히 다른(무관한) 청크가 검색되었습니다. 같은
인물에 대한 질문인데 표현 방식만 바꿔도 검색 결과가 크게 달라진
것입니다.

원인으로 의심되는 지점: `build_rag_prompt_v2`의 주어 추출 정규식
(`re.sub(r"[은는이가]?\s*(누구|뭐|무엇)(야|예요|입니까|인가요)?\??$", ...)`)이
"누구"/"무엇" 패턴만 처리하고 "어떤 업적으로 유명한가" 형태는 처리하지
못해, 질문 전체가 그대로 검색 쿼리로 들어갔을 가능성이 있습니다. 이
경우 임베딩이 "세종대왕"이라는 핵심어보다 질문 문장 전체의 평균
의미에 더 가까운 청크를 찾았을 수 있습니다.

다만 이는 현재 시점의 가설이며, 실제로 정규식이 무엇을 반환했는지
출력해 확인하지는 않았습니다. 추가 실험을 통한 검증이 필요합니다.

→ 평가셋 v2에서는 이 부분을 검증 항목으로 추가하거나, 질문 자체를
검색 쿼리로 쓰는 대신 핵심어만 추출하는 방식을 더 일반화할 필요가
있습니다.

## Evaluation v1 결론

- Q1~Q9는 관련 문서를 대체로 성공적으로 검색했습니다. Retriever는
  예상보다 안정적으로 동작했습니다.
- 생성 모델은 검색된 정보를 의미 있게 요약·정리하는 능력이
  부족했습니다. 9개 중 완전한 성공은 없었고, 키워드 일관성을
  유지한 "부분 성공"이 3건(Q3, Q7, Q9)이었습니다.
- Q10에서는 검색 자체가 실패했습니다 — 질문 표현 방식에 따라
  검색 쿼리 추출이 불안정해질 수 있다는 것을 보여줍니다.
- 현재 가장 큰 병목은 생성 모델의 표현력이지만, 검색 쿼리
  전처리(주어 추출 로직)에도 개선 여지가 있다는 것을 이번 평가로
  추가로 확인했습니다.
